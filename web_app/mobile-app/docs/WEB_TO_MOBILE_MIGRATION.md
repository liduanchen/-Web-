# EPI-Vision Mobile 可视化模块设计与实现

## 概述

EPI-Vision Mobile 的六个核心可视化模块——干涉光谱拟合、Snell 光路仿真、晶圆均匀性分布、原子晶格构型、驻波光场深度解析以及动态色散建模——是系统物理可解释性的关键承载层。移动端受限于 uni-app 框架的 Canvas 图形能力、无 WebGL 支持以及移动设备 GPU 性能差异，无法直接使用桌面 Web 端的前端渲染管线。为此，系统采用了"服务端预渲染为主、客户端 Canvas 绘制为辅"的混合策略：所有复杂图表（光谱、热力图、3D 晶格、色散曲线等）统一由 Flask 后端调用 Matplotlib 生成 PNG 图像并编码为 base64 传输至移动端；仅 Snell 光路仿真这一强实时交互场景保留客户端 Canvas 实现。本章逐一阐述各模块在移动端的实现机制、数据通路与交互设计。

---

## 1. 干涉光谱拟合

干涉光谱拟合是系统最核心的可视化模块，用于将后端反演算法输出的理论拟合曲线、干涉峰标注与原始实验光谱叠加在同一张图上，直观展示模型与数据的契合程度。

移动端采用完全的服务端渲染方案。当用户点击"执行运算分析"后，Flask 后端的 `/api/analyze` 接口在完成 TMM 拟合并返回厚度、置信度等结构化数据的同时，额外调用 `render_spectrum_fit_png_base64()` 函数生成一张完整的干涉光谱拟合图。该函数使用 Matplotlib 的 Agg 后端（无 GUI 依赖）在内存中完成绑制：实验光谱以青色实线搭配浅蓝色面积填充呈现，理论拟合曲线以橙色虚线叠加，干涉峰位置以红色散点标注并加描白色边框以增强辨识度，图表背景色、轴颜色、网格线样式均沿用与 Web 端统一的 Deep Space 深色主题。整张图以 12×6 英寸、100 DPI 的分辨率渲染后编码为 PNG 的 base64 字符串，嵌入响应 JSON 的 `spectrum_plot_b64` 字段一并返回。

移动端接收到 base64 数据后，通过 Vue 的计算属性自动拼接 `data:image/png;base64,...` 前缀，绑定至 `<image>` 组件的 `src` 属性完成渲染。为弥补静态图片缺少交互能力的不足，移动端在光谱图区域实现了完整的手势控制层：通过监听 `touchstart`、`touchmove`、`touchend` 事件，系统实时计算双指间距离相对于初始距离的比例，驱动 CSS `transform: scale()` 实现 1 倍到 6 倍的无级缩放；当图片处于放大状态时，单指拖拽可平移视口，单击则一键复位。这种手势方案虽无法复现桌面端的悬浮数值提示和图例筛选功能，但解决了移动端小屏查看密集光谱曲线的核心痛点。

此外，四个辅助可视化图表——晶圆热力图、晶格构型图、驻波光场图和色散曲线图——也采用完全相同的"服务端渲染 → base64 传输 → `<image>` 展示"的流水线，仅在各自的 API 端点与 Matplotlib 绑制函数上有所区别，详见后续各节。

## 2. Snell 光路仿真

Snell 光路仿真是移动端唯一保留客户端实时渲染能力的可视化模块，因为用户需要连续拖动滑块调节厚度、折射率、入射角等参数并即时看到光路变化，服务端往返延迟无法满足这一实时性要求。

### 2.1 仿真器内嵌光路

仿真器页面（`pages/simulator/simulator.vue`）内嵌了一个光路 Canvas，通过 uni-app 的 `uni.createCanvasContext()` API 进行绑制。该 API 与标准 HTML5 Canvas 2D 在使用模型上有两个关键差异：一是所有绑制操作完成后必须显式调用 `ctx.draw()` 方法将指令提交至渲染层，而非浏览器的立即模式；二是 uni-app Canvas 不支持渐变填充和阴影模糊，因此在渲染介质区域时只能使用纯色加透明度来区分空气层、薄膜层和衬底层。

绑制流程如下：首先将 Canvas 垂直划分为三个区域——顶部约 28% 高度为空气层、中部为薄膜层、底部为衬底层。薄膜层的视觉高度并非按厚度线性映射，而是采用对数缩放公式 `15 + log(1 + t) / log(1 + 100) × 120` 进行计算，使得 0.05 微米到 100 微米的大跨度厚度在有限的 Canvas 画布中都具有合理的视觉比例。然后根据 Snell 定律 `n₀·sinθ₀ = n₁·sinθ₁` 计算折射角，从入射点开始追踪光线在薄膜内部的多次反射路径，每次反射后线条色彩透明度递减一个固定步长（约 0.08），模拟光能随反射次数增加而衰减的物理过程，最多追踪 14 次反射。光线颜色编码为：入射光使用冰蓝色粗实线，内部反射光使用递减透明度的冰蓝色虚线，从表面出射的干涉光使用紫色虚线。每一条光线的坐标均由折射角和薄膜的几何边界通过三角运算精确计算。

参数调节方面，四个核心物理量——外延层厚度、薄膜折射率、衬底折射率和入射角——均同时提供 slider 滑块（触控友好）和数字输入框（精确输入）双通道，任一通道的值变更会通过 Vue 的响应式系统触发 `updateAll()` 方法，继而异步调用 Canvas 重绘。此外，仿真器支持从分析面板导入拟合结果：分析面板通过 `uni.setStorageSync` 将拟合参数和实验光谱数据写入本地存储，仿真器在页面显示时读取并自动同步至各路控件；一旦用户手动调整任何参数，系统标记 `userTouched` 标志位，切换到"参数驱动仿真"模式，允许用户基于拟合结果进行假设分析。

### 2.2 独立光路示意页

除仿真器内嵌光路外，移动端还新增了一个独立的光路示意页面（`pages/optics/optics.vue`），专注于 Snell 定律的教学演示与物理原理验证。该页面提供了五个可独立调节的参数——上层介质折射率、入射角、薄膜折射率、衬底折射率和薄膜厚度——每个参数均配有滑块和数字输入框。

该页面的 Canvas 绑制比仿真器版本更为丰富：除了基础的光线追踪外，还实现了入射角和折射角的圆弧指示标注（在入射点处以灰色细弧线标记角度范围并附数值标签）、全内反射的实时检测与警告（当 Snell 定律计算结果超出定义域时，界面底部显示红色警告文字"全内反射发生"）、以及完整的颜色编码体系——红色实线表示入射光、橙色实线表示表面反射光、绿色实线表示进入薄膜的折射光、绿色半透明虚线表示在衬底界面反射回薄膜的二次反射光、紫色实线表示最终从表面出射的光、青色虚线表示透射进入衬底的光。这种直观的多色光线编码使抽象的电磁波传播过程具象化，适合教学场景和原理验证。

## 3. 晶圆均匀性分布

晶圆均匀性分布模块展示的是基于单点光谱反演厚度推算出的整片晶圆表面膜厚分布预估图。

在移动端，该功能通过新增的 API 端点 `/api/plot/wafer` 实现。Dashboard 页面在收到分析结果后，异步调用该接口，传入当前测算出的核心厚度值和材料名称。后端 `render_wafer_heatmap_png_base64()` 函数在 Python 中使用 NumPy 构建一个 40×40 的二维网格，每个网格点的坐标映射到归一化的 -1 到 1 区间。函数首先通过半径平方判据 `r² ≤ 1` 筛选出晶圆圆形边界内的有效点，然后对每个有效点叠加两层物理形貌特征：一是边缘翘曲效应，通过公式 `bow = r² × 0.03 × thickness` 引入一个抛物面衰减函数，模拟真实化学气相沉积工艺中因边缘气体流速和温度梯度导致的中心厚、边缘薄的宏观特征；二是表面微观粗糙度，通过固定种子的随机数生成器为每个网格点叠加 ±0.001 微米级别的高斯噪声，模拟测量仪器的系统底噪和薄膜表面的真实微观起伏。晶圆圆形边界外的数据点设为 NaN，Matplotlib 在渲染时自动将其处理为透明区域。最后以 `viridis` 色阶映射厚度值并通过双线性插值平滑过渡，附加 colorbar 图例标注厚度数值范围。整张图以 8×7 英寸、100 DPI 渲染为 PNG 的 base64 字符串返回。

移动端将返回的 base64 数据展示在可视化中心的"晶圆均匀性"选项卡内，作为一张静态图片，用户可通过通用的双指缩放手势查看局部细节。

## 4. 原子晶格构型

原子晶格构型模块根据当前分析的材料种类，动态生成对应的三维微观晶格球棍模型，帮助用户从原子排布角度理解材料的结构特性。

移动端通过 `/api/plot/crystal` 端点实现。后端 `render_crystal_lattice_png_base64()` 函数利用 Matplotlib 的 `mplot3d` 三维投影工具包，在一个 3×3×3 的立方晶格框架内，按照棋盘格规则（坐标和 `(x+y+z) % 2 == 0` 的格点为 Type A 原子，其余为 Type B 原子）排布两类原子。原子以散点形式呈现，Type A 使用较大的圆点（s=300），Type B 使用稍小的圆点（s=200），两者均加白色边缘描边以增强视觉立体感。原子之间的化学键以灰色线段连接，分别在 X、Y、Z 三个方向上遍历绘制。

系统为四种预设半导体材料配置了独立的配色方案：碳化硅（SiC）以青色和深灰色分别表示硅原子和碳原子；硅（Si）两种格点均使用冰蓝色；氮化镓（GaN）以紫色表示镓原子、银白色表示氮原子；砷化镓（GaAs）以琥珀色和灰蓝色区分。这与桌面 Web 端 Three.js 版本的材料配色完全一致。三维坐标系的三个轴平面设为透明、轴标签以浅色呈现，保持与系统整体深色主题的协调。渲染完成后以 8×7 英寸、100 DPI 导出为 PNG 的 base64 字符串。需要指出的是，与 Web 端的 Three.js 实时 3D 渲染（支持自动旋转和鼠标拖拽旋转）不同，移动端得到的是从 Matplotlib 默认视角拍摄的单帧静态投影图，无法进行视角交互。这受限于移动端无 WebGL 支持以及 uni-app Canvas 不具备 3D 绑制能力。

## 5. 驻波光场深度解析

驻波光场深度解析模块将抽象的电磁波干涉现象具象化为一张二维热力图，直观展示入射光与反射光在薄膜内部不同深度位置叠加后形成的驻波场强度分布。

在移动端，该模块通过 `/api/plot/standing-wave` 端点实现。后端 `render_standing_wave_png_base64()` 函数接收三组数据：波数轴数组（50 个频点）、深度轴数组（从薄膜表面到底部界面的 50 个深度切片）、以及一个 50×50 的场强二维矩阵。该矩阵由分析算法在反演过程中同步计算生成——对每个深度位置，算法基于当前的薄膜折射率和厚度，沿波数轴计算入射波和反射波的相位差及叠加后的电场强度模值，构成完整的驻波场强度分布数据。

Matplotlib 使用 `pcolormesh()` 函数将该矩阵渲染为伪彩色网格图，色阶选择 `inferno`（从黑色到暗红再到亮黄的渐变），与物理直觉相符——暖色调代表相长干涉区域（场强高），冷暗色调代表相消干涉区域（场强低）。图表附带 colorbar 标注场强相对值，横轴为波数（单位 cm⁻¹），纵轴为薄膜内深度（单位 μm）。整张图以 10×6 英寸的宽幅比例呈现，以容纳深度维度上的更多细节。

移动端在可视化中心的"驻波光场"选项卡中展示这张热力图。通过观察该图，用户可以直观判断在当前膜厚下，目标工作波长的光场能量集中在薄膜的哪个深度位置——这对于需要精确控制有源区位置的光电器件设计具有直接的参考价值。

## 6. 动态色散建模（n-k）

动态色散建模模块展示半导体材料复折射率（折射率 n 和消光系数 k）随波数变化的特性曲线，是评价材料光学性质和分析反演结果可信度的重要依据。

移动端通过 `/api/plot/dispersion` 端点实现。后端 `render_dispersion_png_base64()` 函数利用 Matplotlib 的双轴机制（`twinx()`）在一张图上同时绑制两个不同量纲的物理量：左轴对应折射率 n，右轴对应消光系数 k，两者共享底部的波数横轴。

绑制内容包括四条曲线和一组置信区间：折射率 n 的平滑曲线以冰蓝色（`#38bdf8`）粗实线绑制；消光系数 k 以玫红色（`#f43f5e`）虚线绑制在右轴上；一条理论参考 n 值曲线以灰色点线呈现在左轴上，表示当前材料预设的标准折射率值；围绕 n 曲线上下方各偏移一定幅度（与拟合均方误差相关联）形成置信区间包络，以半透明冰蓝色面积图填充，表征反演结果的不确定度范围。四种线型和双色轴的设计使得多达五个数据维度的信息在一张图中层次分明地呈现。

图表以 10×6 英寸、100 DPI 渲染为 PNG 的 base64 字符串返回。移动端在可视化中心的最后一个选项卡中展示该图，用户可通过双指缩放查看不同波段的细部特征。

## 7. 统一的交互与展示模式

六个可视化模块在移动端的交互范式在整体上是统一的，具体体现在：

**服务端渲染的四类图表**（干涉光谱、晶圆热力图、晶格构型、驻波光场、色散曲线）均遵循"异步 API 请求 → 接收 base64 → 绑定 Vue 数据 → `<image>` 组件渲染"的通路。在移动端展现为一个横向滚动的选项卡栏（`.viz-tabs-scroll`），用户通过点击选项卡在不同视图之间秒切换。每个选项卡对应的图片数据在分析完成后一次性批量加载：主光谱图由分析接口直接返回，其余四个辅助图表通过独立的 `/api/plot/*` 接口异步获取，各自独立请求互不阻塞。在图片数据尚未返回时，对应选项卡显示"渲染中..."占位文字。

**客户端渲染的光路仿真**则不参与选项卡体系，而是独立占据仿真器和光路示意两个完整页面，利用 Canvas 绑制实现实时交互响应，滑块每次值变更均在毫秒级触发重绘。

**图片手势交互**通过 `touchstart`、`touchmove`、`touchend` 三个事件统一处理，适应移动触控场景进行双指缩放与单指平移操作，适用于所有静态图片类的可视化展示。

**视觉主题一致性**得益于 CSS 变量系统：移动端在 `App.vue` 的 `:root` 中声明了 14 个主题变量（涵盖主色调、卡片背景、边框色、文本色等），所有页面组件通过 `var(--xxx)` 引用这些变量，而服务端 Matplotlib 绑制函数中也硬编码了与这些变量对应的十六进制颜色值，在需要时通过颜色映射表将图表元素的颜色参数与服务端渲染的颜色码一一对应，确保前后端视觉体验的高度统一。

**API 设计**统一遵循 RESTful 风格，所有可视化相关的接口均使用 POST 方法并将参数置于请求体中，响应体格式统一为 `{"ok": bool, "image_b64": str}` 或含 `error` 字段的错误格式，由 `common/api.js` 的请求封装层统一处理状态码校验和错误提取，页面层只需关注业务数据。
